Compile and run java program with packages using terminal

09/10/2020
By Maisha Maliha

Today we are going to see how to compile and run java program with package in terminal. If you new to the whole terminal thing I suggest you check my these posts first:

  • How to use terminal to compile and run java program
  • Compile and run java program in different directory using terminal

In this image you can see that we have set the path to the root of the folder, which is “New folder”. From there we compile the “test.java” program.

As you can see we didn’t set any folder name “pack1” inside class folder. Also we only wrote “ -d “class” ” which means we are indicating the "class" file. We told the compiler to create class files inside “class” folder. But the compiler automatically created the “pack1” folder inside "class" folder. Why?? Because this “test.class” is part of “pack1” package.

Every class under “pack1” package wil be kept inside “class/pack1” path. If we had another package, another folder would have been created with its name inside class folder.

Now if we want to run the class file, we have to write:

java –cp “class” pack1.test

Here , we have told where the class file is, which is inside “class” folder. Inside class folder it is inside the “pack1” package folder. That’s why we write “pack1.test”. so the whole thing is:

Java –cp [class path] [packagename].[classname]

If we had written

Java –cp “class/pack1” test

This will show error.

No we are going to create the class file inside the same file as the java program.

Now to run the class file we have to write:

Java pack1.test

Now what if we want to keep the class files inside package but inside another folder in the package.

So we created another folder inside “pack1” package folder by writing “ -d “pack1/class” ”. Now inside the folder “pack1/class” another folder has been created by the compiler as like before. To keep all the class files of a particular package in a particualr folder.

As usual, to run it, you give the path untill package name. And then you write the package name and the classname. See the image below.
